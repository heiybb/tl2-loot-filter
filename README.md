# TL2 Loot Filter (tl2-loot-filter)

A Torchlight 2 loot filter — an injected `d3d9.dll` proxy DLL with an ImGui interface.
It vetoes junk drops before they hit the ground (junk simply vanishes into the void), and gives
you an adjustable filter panel + a recent-drops log. Built in both Chinese and English, reverse-
engineered from `Torchlight2.exe` 1.25.9.5 (AOB signatures are cross-version; also verified on
Steam 1.25.5.6).

> Personal single-player use. **Active filtering is single-player / host-only**; in multiplayer use
> observe-only mode (`logOnly` on, everything else off) — otherwise it may crash.

> 💡 **Strongly recommended: run the game with [DXVK](https://github.com/doitsujin/dxvk).** It is not
> required for this filter to work, but TL2's native D3D9 is crash-prone on modern GPUs (especially
> NVIDIA RTX); DXVK (D3D9 → Vulkan) eliminates most of those crashes, and the ImGui overlay runs
> great on it. Setup: rename DXVK's `d3d9.dll` to `d3d9_dxvk.dll` so it coexists with this filter
> (see *Install / Use* step 4).

## What it does

- **Filter before landing**: hooks the drop-landing point (`sub_5EAA80`); a drop matching your
  rules is never spawned.
  - Block by **UNITTYPE** (e.g. `MAGIC SOCKETABLE`, `POTION`, `SCROLL`, …) *(not fully tested yet)*
  - Block by **name keyword** (substring, case-insensitive) *(not fully tested yet)*
  - Block by **rarity** (white / green+blue / gold / red / purple)
  - Block by **level floor**
  - **Safety switch `logOnly`**: log only, don't actually remove — observe first, enable real
    filtering once you've confirmed the verdicts are correct.
- **ImGui overlay** (toggle with `Home`): rule editor + a "recent drops" table (name / level /
  UNITTYPE / status), with TL2 rich-text color tags `|cAARRGGBB…|u` rendered.
- **Chinese names**: the zh build loads a system CJK font; UNITTYPE + item names show in Chinese.
- **Hook B (optional, experimental)**: don't keep the previous level's ground loot when you leave a
  level (`clearOnTransition`) — revisiting clears it.

## How it works (key design)

- **Injection**: `d3d9.dll` proxy + [MinHook](https://github.com/TsudaKageyu/minhook). At runtime it
  loads the real d3d9 in order: first `d3d9_dxvk.dll` in the game dir (DXVK users rename it to that),
  **else falls back to the system's native `d3d9.dll`**.
  → **No DXVK required**; the `d3d9_dxvk.dll` step only exists so it can coexist with DXVK (both want
  to be named d3d9.dll).
- **Engine location**: everything is found by **AOB (byte-signature) scanning**, no hardcoded RVAs
  (`hooks.cpp` `ResolveEngine`), so it works across versions. If a critical signature doesn't match
  (unsupported version) → it exits safely and installs no hooks.
- **Reads from memory, never from unpacked MEDIA files** (a normal player's MEDIA lives inside
  PAK/MOD, not unpacked):
  - Item name / level / unittype id: read from the in-memory CItem (GUID → DISPLAYNAME, etc.).
  - **UNITTYPE id → English name**: the `kItemUnittypes` table (generated by
    `extract_item_unittypes.py`; equipment UNITTYPEs are fixed base-game definitions with stable
    ids, so no runtime memory read is needed).
  - **Chinese translation**: read directly from the game's in-memory translation table
    (`CStringTranslate`), which reflects the mod load-order overrides actually in effect.

## Files

| File | Purpose |
|---|---|
| `hooks.cpp` | d3d9 proxy + AOB engine location + drop detour + memory reads (unittype/translation) |
| `overlay.cpp` | ImGui overlay (Present hook, panel, color-tag rendering, CJK font) |
| `loot_filter.cpp/.h` | Filter rules + verdict + drop log (shared state) |
| `hook_b.cpp` | Hook B (don't persist ground loot on level transition) |
| `lang.h` | Compile-time i18n macro `LF_T(zh, en)` (`/DLF_EN` selects English) |
| `d3d9.def` | Export table (proxy forwarding) |
| `extract_item_unittypes.py` | Generates the `kItemUnittypes` whitelist (scans ITEMS + UNITTYPES.HIE) |
| `third_party/` | imgui + minhook (vendored) |

## Build

Requires **MSVC (VS 2022 / v143) + Win32**. The project **builds both zh and en**, and copies the
`DeployLang` build (default `en`) into the game directory.

```bat
:: after locating MSBuild via vswhere:
msbuild tl2_loot_filter.vcxproj /t:Build /p:Configuration=Release /p:Platform=Win32 /p:DeployLang=zh
```

- Outputs: `bin\Release-zh\d3d9.dll`, `bin\Release-en\d3d9.dll`.
- **Deploy target** is overridable: `/p:GameDir="your TL2 dir"` (default `E:\Torchlight 2\`; if the
  dir doesn't exist, deployment is skipped automatically).
- Language: `/p:DeployLang=zh` (Chinese) or `en` (English).

## Install / Use

1. Put the `d3d9.dll` of your chosen language (zh or en) into the **game root** (next to
   `Torchlight2.exe`).
2. Launch the game; toggle the filter panel with **`Home`**.
3. Turn on `logOnly` first and watch the "recent drops" table to confirm the verdicts; then turn
   `logOnly` off to enable real filtering.
4. **DXVK (strongly recommended)**: rename DXVK's `d3d9.dll` to `d3d9_dxvk.dll` and drop it in the
   game root alongside this filter's `d3d9.dll` — the proxy chains to it automatically. Greatly
   improves stability on modern / NVIDIA systems.

## Safety notes

- Active filtering (veto-on-landing) = **single-player / host only**. Enabling real filtering on a
  multiplayer client is pointless and may desync.
- If a critical AOB doesn't match (unsupported game version) → the DLL installs no hooks and doesn't
  touch the game.
- All memory reads are wrapped in SEH (`__try`); a bad pointer won't crash.
